Overview

In the world of Web 2.0, applications developed by different companies don't talk to each other. A meeting invitation you received on Outlook doesn't appear on your Google Calendar and your search preferences for Google are not used if you search on Bing. The reason behind this is that different companies build their own ecosystem and don't want to share, import and export functionalities across platforms that they builds on.

Going into the Web 3.0 era, applications are built on a common platform - blockchain, where each decentralized application is built by following similar if not exactly the same standards, which makes the data and functionalities easily shareable between applications. One blockchain application can access and modify the data and state of another application without much obstables. The barriers Web2 users used to face no longer exist in the Web 3.0 world where users are capable of doing more interesting stuff by exploring this interoperability nature of blockchain and Web 3.0 applications.

Neverthelss, although blockchains and applications on them are public accessible, interacting with blockchain or smart contracts is not trivial task for everyone. Moreover, each blockchain user might have different needs when using blockchain, especially considering the vast amount of interoperable applications on blockchains. For certain activities, it might involve monitoring state of some protocols, and then make some trades on another protocol while later send the received tokens to another protocol for other interactions. Currently, most personalized interactions have to be coded by experiences developers, even by using smart contract automation platforms such as Gelato Networks that offers some level of abstraction. This process is very similar to how application development works in Web 2.0 era that requires time, expertise and costs money and the results can still fall short of the personalized requirements by users due to the limits posed by existing solutions.

KliK's mission is to make personalized interactions with blockchain both easy and cheap enough for every blockchain user, whether it's just for simple automation tasks such as scheduling transactions or listening to events, or much more complicated tasks that involve a lot of steps in the execution. We are achieving this goal by adopting the following approaches:

  • A No-Code, Reconfigurable graphic interface for users to design, organize and build the workflows they need for interacting with blockchain and smart contracts. By creating no-code and play-and-plug tools for users, they can quickly build personalized executable workflows without deep technical knowledge of coding or blockchain. Moreover, this approach also makes it possible to satisfy almost every possible use cases by simply reconfiguring different on-chain and off-chain components for workflows of various complexity. As there are Legos for infants, but also for teenagers.

  • A network of Generative AI Agents that are trained for blockchain workflow designs by receiving inputs such as simple commands or conversations. Generative AIs have been proved to show strong reasoning capabilities once well trained or fine-tuned. They are the perfect tool to help users navigate difficult problems such as workflow design even though they don't have strong technical background or domain knowledge.

  • A network of Automation Agents that are able to execute the workflow designed either by users or AIs. They will host the workflow using the execution stack built into KliK and their node infrastructure. These agents will send back necessary execution results to users for alerts, visualization, analysis and transaction approval.

Our Vision

As a public infrastructure that is accessible by everyone, blockchain should be made easy to use. Everyone should be able to build a simple blockchain application quickly enough in order to best utilize the benefits that blockchain offers. However, every blockchain user has their individual needs, personal requirements, technical expertise, required complexity of the tasks, etc. This would lead to applications of very distinct features and functionalities to be built and customized. For simple use cases such as scheduling transactions or placing limit order on AMMs, to certain very complicated use cases that requires interactions with multiple smart contracts, various trigger conditions, combined with complicated computational tasks.

Our vision is an ambitious one. We want to have a platform that can be used to build blockchain applications based on ANY request a user might have, thus solving this problem all at once. Any blockchain user can come to the platform with some request for doing something using blockchain and we will give the user an application that can help the user achieve the goal to its best. The user doesn't have to be a developer, or have very deep understanding of blockchain and smart contracts, as long as the user can express the intention clearly, the application can be built.

Certainly to solve this problem at scale, you can't just employ an army of developers and build an application whenever a user comes. You must employ a very good framework that can be generalized enough to common use cases, but also tailored enough to satisfy the needs for every request that user has. Luckily, we have identified the following techniques that can help us accomplish our ambitious vision.

  • Abstraction. Blockchain applications are typically highly logical and have limited scope. They usually involve performing tasks step by step by utilizing existing funcionalities provided by smart contract. Employing abstraction technique to present an application as a workflow would help with our vision. Each workflow would simply be a different combinations of smaller components and modules that can perform smaller tasks such as computation and smart contract interactions.

  • Generative AI. Another technique that can help us achieve the goal is to employ generative AI when fulfilling user requests. Generative AIs have shown strong reasoning capabilities through good understanding of human intentions and it keeps improving over time when newer generations of LLMs. AIs can be trained to understand verbal requests from users, extracting necessary components for fulfilling the requests and reason about the structure for the application by configuring the components. Therefore, an army of Web3 developers can be replaced with a powerful AI agent to help users design blockchain applications according to their individual needs.

  • Decentralization. Executing blockchain applications require certain infrastructure such as nodes and computation resources. With the development of decentralization, this makes it possible to scale execution easily by connecting users to a network of decentralized execution agents. The users can send the generated workflow of the application to any available agent, which will employ necessary resources to execute the application.

  • Crypto. Incentivisation is the key to successful bootstrapping of the platform we aim to build. It helps both acquire important data to fine-tune and train generative AIs in the blockchain domain, but also to incentivise node operators as agents to participate in the execution network. Crypto is the best option for bootstrapping such participation where blockchain users and node operators can get paid by providing either high quality training data or execution service.

Considering everything above, our vision and roadmap at KliK Networks going forward become extremely clear. We will create a platform that designs and executes personalized blockchain applications by designing and building state-of-the-art technologies that involve blockchain abstraction, generative AIs, and decentralized economies.

System Design

Workflow Design

Blockchain users can use a visual workflow design GUI on KliK to create workflows according to their personal needs. Users can easily search and import components connected to smart contracts, blockchain transactions or generic operators into the GUI and build workflows by connecting and configuring these components.

After workflows are designed and saved, they will be stored on blockchain that can be accessible by both other users as well as AI agents. Other users can reuse the same or similar workflow for their own needs, while AI agents can use the saved workflow data to train or fine-tune their model to better generate useful workflows for other users in the future.

Generative AI

When AI agents are trained and fine-tuned, they will be able to understand blockchain knowledge and automatically generate workflows based on users' inputs through simple methods such as chats and conversations. The generated workflows by AIs will render in the same GUI for users to inspect and configure. If they find something is wrong or not perfect, they can always make changes before pushing the workflow to automation agents for execution, especially the workflow involves financial transactions so that mistakes made by AIs can be found at this stage.

Automation

Once a personal workflow is either designed by the user or generated by AIs, the workflow can be automated by leveraging the automation agents in the network. The workflow data is sent to an available automation agent and the automation agent will start executing the workflow by connecting necessary data sources, node interfaces, etc. Troughout the execution process, data, alerts or transaction approval requests will be forwarded to the user of this workflow.
